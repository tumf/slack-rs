# 提案: コマンド出力の統一と raw 出力オプション

## 概要
`api call` とラッパーコマンドの JSON 出力構造を統一し、共通の `response`/`meta` エンベロープで返す。互換性のために `--raw` を追加し、必要に応じて従来の Slack API そのままのレスポンスを返せるようにする。

## 背景
現在、`api call` は `response`/`meta` で包むが、ラッパーコマンドは Slack API そのままを返している。スクリプトや `jq` の利用者がコマンドごとにパスを変える必要があり、学習コストと運用コストが高い。

## 目的
- すべてのコマンドで JSON 出力の形を統一する
- 実行コンテキスト（profile/team/user/method/command）を機械的に取得できるようにする
- 互換性のために raw 出力を選択可能にする

## 非目的
- Slack API のレスポンス内容やパラメータを変更すること
- 人間向けの表示フォーマット（表、CSV など）の追加
- 既存 API 呼び出しの挙動や認可方式の変更

## 変更範囲
- `api call` の出力に `meta.command` を追加
- ラッパーコマンドの JSON 出力を `response`/`meta` に統一
- `--raw` で従来の Slack API レスポンスを返す

## 互換性と移行
- 既存の `jq` スクリプトが影響を受けるため、`--raw` を移行用に用意する
- 既存利用者向けに README/ヘルプに移行の注意点を記載する

## リスク
- デフォルト出力が変わる場合の破壊的影響
- コマンドごとの `command` 値の統一が必要
