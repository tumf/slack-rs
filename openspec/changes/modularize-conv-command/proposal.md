# 提案: conv コマンド実装のモジュール分割

## 背景
`src/commands/conv.rs` が巨大化しており、フィルタリング、ソート、フォーマット、API 呼び出しが混在しています。保守性が低下しています。

## 目的
- conv コマンド関連の責務を分割し、見通しと変更容易性を改善する
- CLI 挙動は維持する

## 範囲
- `conv` 関連の関数を責務別に分割（例: filter/sort/format/api）
- re-export により既存の公開 API を維持

## 非目標
- `conv` コマンドの仕様変更
- 出力やフィルタ条件の変更

## 影響・リスク
- 公開関数の可視性やモジュールパス変更による回帰

## 受け入れ基準
- `conv list`/`conv select`/`conv history` が既存と同じ挙動を維持する
- `commands::` からの呼び出しが同じパスで成立する

## 依存関係
- なし
