# 提案: file download の回帰テスト範囲を拡張する

## 概要
Issue #21 として、`file download` の `file_id` 指定と `--url` 指定の双方で回帰を固定する。過去に `files.info invalid_arguments` が発生した経路の再発を防ぐため、認証付きダウンロードの主要経路を仕様とテストで明確化する。

## 背景
- 既存仕様は `file download` の基本挙動を定義しているが、入力モード別の回帰観点が明示的でない
- 実運用では `file_id` と `--url` の両方が使われるため、どちらか一方のみの検証では不十分

## 目的
- `file_id` と `--url` の両方が認証付きダウンロード経路に接続されることを固定する
- `files.info` 呼び出し時の引数形式ミス再発を防ぐ

## 非目的
- `file download` の新オプション追加
- ダウンロード結果フォーマットの変更

## 提案内容
- `wrapper-commands` 仕様に入力モード別シナリオを追加する
- モックベースの統合テストで `file_id` と `--url` の双方を検証する

## 期待効果
- 過去不具合の再発防止
- CI での早期検知精度向上
