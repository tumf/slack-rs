## 目的
`file download` が `files.info` を呼ぶ際のパラメータ送信形式を Slack API の期待に合わせ、`invalid_arguments` を解消する。

## 設計方針
- **送信形式の是正**: `files.info` の `file_id` は JSON ボディではなく、フォームまたはクエリとして送る。
- **影響範囲の最小化**: 変更は `file download` の `files.info` 呼び出し部に限定し、後続のダウンロード手順は変えない。
- **再発防止**: リクエスト構築を検証するモック/スタブテストを追加し、JSON ボディ送信を検知できるようにする。

## 実装アプローチ
- `files.info` リクエスト生成で `Content-Type` と送信データの組み立てを見直す。
- テストでは HTTP モックで受信パラメータを検証し、`file_id` がフォーム/クエリに含まれることを確認する。
- 併せて、JSON ボディの `file_id` を要求するスタブ条件を置かずに成功することを確認する。

## リスクと対策
- **リスク**: 送信形式変更により既存テストが前提不一致になる。
- **対策**: 既存テストの期待値を API 仕様に合わせて更新し、リクエスト断面（ヘッダー/ボディ）を明示的に検証する。
