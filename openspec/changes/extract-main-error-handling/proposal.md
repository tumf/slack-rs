# 変更提案: main のエラー終了処理を共通化する

## 背景
- `src/main.rs` では `eprintln!` と `std::process::exit` の組み合わせが多数重複している。
- 分岐追加時に終了コードの不整合が入り込みやすく、保守コストが高い。

## 目的
- 失敗時の出力文言と終了コード処理を小さな共通ヘルパーへ抽出し、一貫性を高める。
- 既存挙動は変更せず、内部構造のみ改善する。

## 変更範囲
- `main` に存在する重複エラー終了パターンの抽出方針を定義する。
- 非対話エラー時の終了コード 2 を含む既存規約維持を要件化する。

## 非目的
- コマンド仕様の追加・変更は行わない。
