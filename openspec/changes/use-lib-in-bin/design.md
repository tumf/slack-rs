# 設計: バイナリからライブラリを利用する構成へ統一

## 方針
- `src/main.rs` は CLI のエントリとして最小限の役割に留め、機能は `slack_rs` ライブラリ側へ委譲する
- `mod api;` などのモジュール宣言を削除し、同一ソースの二重コンパイルを避ける
- 既存 CLI の挙動は変えず、参照経路のみを変更する

## 変更対象
- `src/main.rs` のモジュール宣言と `use` 句
- `src/lib.rs` の公開モジュール（既存の `pub mod` を活用）

## 期待される効果
- `cargo test` 実行時に lib と bin で同一テスト群が重複しない
- バイナリとライブラリの構成が一貫し、保守性が向上する

## リスクと対策
- バイナリから参照するシンボルが `pub` でない場合にビルドエラーが発生する
  - 必要なものはライブラリ側で公開し、`src/main.rs` は `slack_rs::` 経由で参照する
