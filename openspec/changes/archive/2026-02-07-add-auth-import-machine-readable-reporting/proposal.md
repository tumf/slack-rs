# 提案: auth import の競合結果を機械可読に報告する

## 概要
Issue #22 として、`auth import` 実行時に更新・スキップ・上書きの結果を明示的に出力できるようにする。`--yes` / `--force` 時も結果の追跡可能性を担保し、自動化で扱える `--json` 出力を追加する。

## 背景
- 現行仕様は衝突時の挙動は定義しているが、実行後サマリの機械可読性が弱い
- 運用や監査では、どの profile がどう扱われたかの記録が必要

## 目的
- `auth import` 結果を profile 単位で可視化する
- CLI 自動化で処理結果を取り込みやすくする

## 非目的
- import 暗号化フォーマットの変更
- profile 解決ロジックの変更

## 提案内容
- テキスト出力で更新・スキップ・上書き件数と対象を表示する
- `--json` 時は profile 単位の結果配列を返す

## 期待効果
- 運用時の説明責任と監査性の向上
- CI/バッチでの分岐処理が容易になる
