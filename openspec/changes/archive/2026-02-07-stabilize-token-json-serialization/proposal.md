# 提案: tokens.json のシリアライズを安定化する

## 概要
Issue #24 として、`tokens.json` のキー順序とシリアライズ結果を決定的にし、意味的に同一内容で不要な差分が発生しないようにする。

## 背景
- 現行では保存順により JSON 出力順が揺れる可能性がある
- 差分監視やチェックサム運用でノイズが増える

## 目的
- 同一内容なら同一バイト列に近い安定出力を担保する
- 不要な差分と churn を抑える

## 非目的
- 暗号化や保存先パスの変更
- トークンキーの意味変更

## 提案内容
- `tokens.json` 保存時のキー順序を決定的にする
- 内容不変時の再保存で差分が出ないことを回帰テストで固定する

## 期待効果
- レビュー時のノイズ削減
- 自動運用（同期・バックアップ）での安定性向上
