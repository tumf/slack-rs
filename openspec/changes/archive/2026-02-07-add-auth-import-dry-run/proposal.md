# 提案: auth import に dry-run を追加する

## 概要
Issue #23 として、`auth import --dry-run` を追加し、実際の書き込みを行わずに変更予定と競合解決方針を確認できるようにする。

## 背景
- 本番環境や移行前確認で、適用前の影響把握ニーズが高い
- 現行は実行しないと結果が分からず、安全確認のステップが不足している

## 目的
- 書き込みなしで import の影響範囲を事前確認できるようにする
- 競合時の action（更新/スキップ/上書き予定）を明示する

## 非目的
- 既存 import の暗号化・復号処理の変更
- dry-run 実行時の副作用を許容する設計

## 提案内容
- `auth import --dry-run` は設定・トークンファイルへの書き込みを行わない
- 通常出力と `--json` の両方で予定 action を返す

## 期待効果
- 本番反映前の安全性向上
- 移行計画のレビュー容易化
