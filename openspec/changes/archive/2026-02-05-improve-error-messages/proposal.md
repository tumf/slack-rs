# 提案: Slack API エラーのガイダンス追加

## 概要
Slack API から `ok=false` が返る場合に、代表的なエラーコードへ原因と解決策のガイダンスを付与する。JSON 出力は維持しつつ、必要な情報を機械可読かつ人間可読の形で補助する。

## 背景
現在のエラー出力は Slack のエラーコードのみで、原因や解決手順が分からない。特にトークン種別やスコープ不足による失敗時にデバッグが困難。

## 目的
- 代表的なエラーに対して原因と解決策を提示する
- JSON 出力の互換性を維持しながら補助情報を付与する
- スクリプトと人間の両方で利用できる情報設計にする

## 非目的
- すべての Slack エラーコードの網羅
- 既存 API の挙動や認可方式の変更
- 外部システムへの追加依存

## 変更範囲
- 代表エラー（例: `not_allowed_token_type`, `missing_scope`, `invalid_auth`）にガイダンスを付与
- `api call` とラッパーコマンドで同じガイダンス方針を採用

## リスク
- ガイダンスが古くなる可能性
- エラー出力の増加によるログ量の増加
