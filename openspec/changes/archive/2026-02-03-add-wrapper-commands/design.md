# 設計: ラッパーコマンドと安全装置

## ラッパー戦略
- `api call` を内部的に使い、入力の整形とデフォルト値のみを提供する
- 出力は `api call` と同様に meta を含む JSON を返す

## write 安全装置
- write コマンドは必ず `--allow-write` を要求する
- 破壊操作は `--yes` で確認をスキップできる
- `--yes` が無い場合は対話的確認を行う

## 入力検証
- channel id / user id / timestamp / emoji は簡易パターンで検証する

## テスト方針
- モック HTTP サーバで各コマンドのリクエストを検証
- write ガードはユニットテストで検証
