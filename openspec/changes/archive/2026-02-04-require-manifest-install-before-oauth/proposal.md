# 提案: OAuth 認証前にマニフェストをインストールさせる

## 背景
現在の `auth login` は OAuth 認証の完了後にマニフェストを生成・保存しているため、Slack アプリ側の設定が未完了のまま認証フローが開始される。これにより、ユーザーがマニフェストを事前にインストールする必要がある運用に適合しない。

## 目的
OAuth 認証を開始する前にマニフェストを生成し、Slack アプリ管理ページでユーザーがインストールできるように誘導する。

## 変更概要
- `auth login` で OAuth 設定を解決後、認証開始前にマニフェストを生成・保存する。
- マニフェストの保存パスを提示し、Slack App 管理ページをブラウザで自動的に開く。
- ユーザーがマニフェストをインストールしたことを確認してから OAuth 認証フローを開始する。

## 対象範囲
- `auth login` の実行フロー
- マニフェスト生成タイミングとユーザー案内

## 対象外
- Slack App 管理ページでの実際のインストール作業（ユーザー操作）
- OAuth 認証 URL 生成やトークン交換の仕様変更

## リスク・考慮事項
- マニフェスト生成・案内の手順が増えるため、CLI の対話手順が長くなる可能性がある。
- ブラウザ自動起動が失敗した場合のフォールバック表示が必要。
