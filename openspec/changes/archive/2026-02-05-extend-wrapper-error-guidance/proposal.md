## 変更概要
Issue #10 の「エラーメッセージ改善」に対応し、ラッパーコマンド（`conv`/`users`/`msg`/`react`/`file`/`search`）で既知の Slack エラーガイダンスを表示する。

## 背景
- `api call` では既知エラーに対するガイダンスが表示されるが、ラッパーコマンドでは一貫していない。
- どのトークンを使うべきか、どのスコープが必要かが分からず、トラブルシュートが難しい。

## スコープ
- ラッパーコマンドで `ok=false` かつ既知エラーコードが返った場合にガイダンスを標準エラー出力へ表示する。
- JSON 出力の内容は変更しない。

## スコープ外
- エラーガイダンス文面の追加/編集（既存の `api/guidance` を再利用）。
- 既知エラーの網羅性拡充。

## 既知のリスク
- 標準エラー出力が増えるため、既存のログ処理に影響する可能性がある。

## 受け入れ基準
- `conv`/`users`/`msg`/`react`/`file`/`search` の各ラッパーコマンドで、既知エラー時にガイダンスが表示される。
- JSON 出力の構造は従来通りである。
