## 目的
ラッパーコマンドでも `api call` と同等のエラーガイダンスを表示し、原因と対処を理解しやすくする。

## 設計方針
- **再利用**: 既存の `api/guidance` と `display_error_guidance` を利用する。
- **出力分離**: JSON 出力は変更せず、追加情報は標準エラー出力に限定する。
- **共通化**: 各コマンドの重複を避けるため、ガイダンス表示を共通関数にまとめる。

## 具体的な挙動
- ラッパーコマンドが Slack API 応答を受け取った直後にガイダンス表示処理を呼ぶ。
- `ok=false` かつ既知の `error` コードのみガイダンスを出す。

## 代替案
- 各コマンドに個別実装する案は重複が多いため採用しない。
