# 設計: auth login で既定トークン種別を保存

## 方針
- `auth login` の保存処理に、既定トークン種別の初期化を追加する。
- 既存の優先順位と環境変数の挙動は変更しない。

## 変更点
- OAuth 交換結果から取得できたトークンを基に既定値を決める。
  - `authed_user.access_token` がある場合は `user` を既定とする。
  - ない場合は `bot` を既定とする。
- `profile.default_token_type` が既に設定されている場合は、その値を維持する。
- 判定ロジックは小さな純関数に切り出し、ユニットテストで検証可能にする。

## 代替案
- `auth status` の表示を「設定値のみ」に変更する。
  - 既存の表示期待を変更し、情報量が減るため採用しない。

## 影響と互換性
- 既定トークン種別の初期値が保存されるようになるが、明示設定がある場合は既存挙動と同じ。
