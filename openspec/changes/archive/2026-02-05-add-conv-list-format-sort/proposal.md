# 提案: `conv list` にフォーマット/ソート機能を追加

## 概要
`conv list` の結果表示に対して、用途に応じた出力フォーマット（例: `json`/`jsonl`/`table`/`tsv`）と、クライアント側ソート（例: `name`/`created`/`num_members`）を追加する。

## 背景
現状の `conv list` は Slack API（`conversations.list`）の返却順に依存し、取得結果をスクリプトへ渡す際に追加の整形やソートが必要になる。特に以下が手間になりやすい。

- 端末での目視確認（チャンネル名順に見たい、列で見たい）
- パイプ処理での利用（TSV/JSONL で 1 行 1 レコードにしたい）
- 参加状態/公開・非公開のフィルタ後に、一定の順序で一覧化したい

## 目的
- `conv list` の結果を CLI 側で整形できるようにし、外部ツール依存を減らす
- ソート基準を明示できるようにし、結果の再現性を上げる
- 既存のデフォルト挙動（互換性）を維持する

## 非目的
- `conversations.list` のパラメータ追加や API 側ソートの変更
- チャンネル一覧のキャッシュや永続化
- フィルタ仕様の追加・変更（既存の `--filter` は本変更の対象外）

## 変更範囲
- `conv list` に `--format` を追加し、出力形式を選択可能にする
- `conv list` に `--sort` と `--sort-dir` を追加し、取得結果に対してソートを適用できるようにする
- `--raw` は JSON 出力時（デフォルト/`--format json`）のみ有効とし、`jsonl`/`table`/`tsv` との併用はエラーにする
- 不正な値が指定された場合は、ユーザーが修正しやすいエラーを返す

## 互換性
- `--format`/`--sort`/`--sort-dir` 未指定時は、従来の `conv list` 出力（JSON）と同等の内容/順序を維持する
- `--raw` を JSON 以外の出力形式と併用した場合はエラーとなる（出力フォーマットの整合性のため）

## リスク
- `table` 出力は列幅や端末幅で見た目が変わるため、スクリプト用途は `tsv`/`jsonl` を推奨する必要がある
- `num_members` など一部フィールドはワークスペース設定や権限により欠落する可能性があるため、欠落時の扱い（例: 0 扱い）を定義する必要がある
