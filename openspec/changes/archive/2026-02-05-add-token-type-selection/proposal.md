# 提案: トークン種別の明示選択と既定設定

## 概要
`slack-rs` で User Token / Bot Token を明示的に選択できるようにし、プロファイルに既定のトークン種別を保持します。`api call` とラッパーコマンドの両方で `--token-type` を受け付け、出力メタ情報に使用トークン種別を含めます。

## 背景
同一アプリに User Token と Bot Token が共存するケースで、コマンド実行時に意図したトークン種別を選べないため、権限差による結果の違いが分かりづらい状態です。

## 目的
- コマンド実行時にトークン種別を明示指定できる
- プロファイル単位で既定トークン種別を持てる
- 実行結果に使用トークン種別が明示される

## 非目的
- OAuth フローやトークン保存形式の大幅な変更
- 新しい認証方式や外部サービス連携の追加

## 成果物
- `api call` とラッパーコマンドの `--token-type` 対応
- 既定トークン種別の設定・保存
- `api call` の `meta.token_type` 出力

## 依存関係・影響
- トークン解決ロジックの共通化が必要
- 既存プロファイルに `default_token_type` が追加される

## リスク
- 既定の選択ロジックにより従来の挙動が変わる可能性
