# 設計: トークン種別選択

## 前提
- Bot/User トークンは既に別キーで保存可能
- 明示指定がない場合の既定挙動が必要

## 解決方針
### トークン種別の解決順序
1. `--token-type` が指定されていればそれを最優先
2. プロファイルに `default_token_type` があればそれを使用
3. 既定値として `bot` を選択し、存在しない場合は `user` にフォールバック

### 出力メタ情報
`api call` 出力の `meta` に `token_type` を追加し、使用トークンを明示する。

### 既定トークン種別の設定
`config` サブコマンドでプロファイルに `default_token_type` を保存する。
保存先は `profiles.json` とし、既存プロファイルの後方互換性を保持する。

## 代替案
- 既定値を `user` にする案: 既存の bot ベースの挙動と互換性が下がるため採用しない
