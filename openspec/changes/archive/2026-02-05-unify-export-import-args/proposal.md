# 提案: export/import 引数パースの共通化

## 背景
`auth export` と `auth import` の引数パースロジックが重複しており、保守コストとバグの温床になっています。

## 目的
- export/import の引数パースを共通化し、重複を削減する
- CLI 挙動とエラーメッセージを維持する

## 範囲
- 共通の引数パーサー（例: `ExportImportArgs`）を導入
- `auth export` と `auth import` から共通パーサーを利用

## 非目標
- オプション仕様やフラグの追加/変更
- 出力形式の変更

## 影響・リスク
- 引数の解釈順序やデフォルト値の差分が生じる可能性

## 受け入れ基準
- 既存のフラグが同じ意味で解釈される
- 既存の警告/確認/エラー表示が変わらない

## 依存関係
- なし
