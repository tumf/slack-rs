# 提案: バイナリからライブラリを利用してテスト重複を解消

## Why

`cargo test` が同じテスト群を 2 回実行しており、開発者の混乱と実行時間の増加を招いている。これは `src/main.rs` が独自にモジュール宣言を持ち、ライブラリと同じコードをバイナリ側でもコンパイルしているために発生している。バイナリはライブラリの公開 API を利用する構成に統一し、テストの重複を解消する必要がある。

## What Changes
- `src/main.rs` から `mod api;` などのモジュール宣言を削除し、`slack_rs` ライブラリを利用する
- `use` を `slack_rs::...` へ置き換え、バイナリ側に重複モジュールを持たせない

## 影響範囲
- 影響ファイル: `src/main.rs`（参照の整理）
- 外部 API / CLI 仕様への影響なし
- ビルド/テスト出力の重複が解消される
