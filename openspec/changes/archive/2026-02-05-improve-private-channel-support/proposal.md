# 提案: プライベートチャンネル向けのトークン選択とガイダンス

## 概要
`conversations.list` で `types=private_channel` が指定された場合、User Token を優先して使用し、取得できない場合は改善されたガイダンスを表示する。

## 背景
Bot Token では参加していないプライベートチャンネルが取得できず、原因が分かりづらい。

## 目的
- `private_channel` 指定時の期待されるトークン種別の自動選択
- プライベートチャンネル取得失敗時の明確なガイダンス

## 非目的
- トークン保存方式の変更
- 会話リストの高度なフィルタリング（別提案で扱う）
