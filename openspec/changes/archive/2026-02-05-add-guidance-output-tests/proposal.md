## 変更概要
既知の Slack エラーに対するガイダンス表示をユニットテストで保証できるようにし、TDD で仕様を固定する。

## 背景
- ラッパーコマンドのガイダンス表示は仕様化されているが、現状のテストは「panic しない」確認に留まっている。
- stderr 出力を直接検証しづらいため、決定的なガイダンス文字列を返す関数が必要になる。

## スコープ
- ガイダンス生成の純関数を追加し、既存の表示関数がそれを利用する形に整理する。
- 既知エラー/未知エラー/成功ケースのユニットテストを追加する。
- JSON 出力内容は変更しない。

## スコープ外
- ガイダンス文言そのものの改訂。
- ラッパーコマンドの出力フォーマットの変更。

## 既知のリスク
- ガイダンス文字列のフォーマットを固定することで、将来の文言変更時にテスト更新が必要になる。

## 受け入れ基準
- 既知エラー時のガイダンス文字列が `Error:`/`Cause:`/`Resolution:` を含むことがテストで保証される。
- 未知エラー/成功レスポンスではガイダンスが生成されないことがテストで保証される。
