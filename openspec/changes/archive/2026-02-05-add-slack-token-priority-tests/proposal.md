## 変更概要
ラッパーコマンドのトークン解決で `SLACK_TOKEN` を最優先する仕様を、TDD のユニットテストで固定する。

## 背景
- 仕様では `SLACK_TOKEN` が設定されている場合は token store より優先することが求められる。
- 現状は token store 優先の実装があり、仕様と挙動が一致しない可能性がある。

## スコープ
- `SLACK_TOKEN` 優先のトークン解決を純関数化し、ユニットテストで検証できるようにする。
- 既存の `--token-type` / `default_token_type` の優先順位は維持する。

## スコープ外
- `api call` のトークン解決ロジックの変更。
- OAuth 設定やトークン保存方式の変更。

## 既知のリスク
- 環境変数の優先度が明確になるため、意図せず `SLACK_TOKEN` を使うケースが顕在化する可能性がある。

## 受け入れ基準
- `SLACK_TOKEN` が設定されている場合、ラッパーコマンドは token store の内容に関係なくそのトークンを使用する。
- `SLACK_TOKEN` が未設定の場合は従来の token store 解決を維持する。
