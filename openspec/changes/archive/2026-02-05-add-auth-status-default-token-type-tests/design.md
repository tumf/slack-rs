## 目的
`auth status` の既定トークン種別表示をテスト可能にし、仕様に沿った表示ロジックを保証する。

## 設計方針
- **純関数化**: 表示に必要な情報を組み立てる関数を追加し、`default_token_type` の優先順位を明示する。
- **後方互換**: 既存の表示項目は維持し、必要最小限の整理に留める。

## テスト戦略
- プロファイルに `default_token_type` が設定されている場合はその値が表示されることを検証する。
- `default_token_type` が未設定の場合のみ、user token 有無による推測が使われることを検証する。
