## 変更概要
`auth status` の `default_token_type` 表示を仕様通りに固定するため、TDD でユニットテストを追加する。

## 背景
- 仕様では `default_token_type` が設定されている場合はその値を表示し、未設定時のみ推測ロジックを使う。
- 現状の実装は token の有無で推測しており、仕様とずれる可能性がある。

## スコープ
- `auth status` の表示ロジックを純関数化し、ユニットテストで検証できるようにする。
- `default_token_type` の優先表示と未設定時の推測ロジックをテストで固定する。

## スコープ外
- `auth status` の表示項目追加（Token Store 表示など）。
- OAuth 設定やトークン保存方式の変更。

## 既知のリスク
- 表示ロジックの分離に伴い、出力の行順や整形が変わる可能性がある。

## 受け入れ基準
- `default_token_type` が設定されている場合、`auth status` はその値を表示する。
- `default_token_type` が未設定の場合のみ従来の推測ロジックを使用する。
