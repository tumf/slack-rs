# 変更提案: auth login の引数解析責務を分離する

## 背景
- `run_auth_login` は引数解析、非対話制約判定、入力プロンプト分岐、実行経路選択を同時に担っている。
- オプション追加時に分岐密度が増え、回帰が起きやすい。

## 目的
- `auth login` の入力解釈を専用パーサ構造へ分離し、実行ロジックと責務を分ける。
- `--cloudflared` / `--ngrok` の排他、非対話制約、既定値適用の挙動を明示固定する。

## 変更範囲
- `run_auth_login` の前段処理を分離するリファクタ方針。
- 既存 CLI 仕様維持を要件として明文化。

## 非目的
- OAuth 認証フロー自体やマニフェスト生成仕様の変更は行わない。
