# 変更提案: auth login の入力解析と OAuth 実行コアを統合整理する

## 背景
- `run_auth_login` は引数解析、非対話制約判定、入力プロンプト分岐、実行経路選択を同時に担っている。
- `auth::login` と拡張ログイン経路には OAuth 実行手順の重複があり、保守時の差分漏れリスクがある。

## 目的
- `auth login` の入力解釈を専用パーサ構造へ分離し、実行ロジックと責務を分ける。
- `--cloudflared` / `--ngrok` の排他、非対話制約、既定値適用の挙動を明示固定する。
- 標準ログインと拡張ログインで重複する OAuth 実行手順を単一コアへ統合する。

## 変更範囲
- `run_auth_login` の前段処理（解析・検証）を分離するリファクタ方針。
- 認証 URL 生成からトークン交換までの実行コア重複を一本化する方針。
- 既存 CLI 仕様・保存結果互換を要件として明文化。

## 非目的
- OAuth パラメータやマニフェスト生成仕様の外部挙動変更は行わない。
