# 設計メモ: auth login 引数解析の分離

## 方針
- 入力解析結果を `LoginArgs` のような内部構造体へ集約する。
- 実行フェーズは解析済み構造を受け取り、標準/拡張ログイン経路を選ぶだけにする。

## 主要ポイント
- 解析フェーズで以下を完結させる:
  - `--cloudflared` / `--ngrok` の排他検証
  - スコープ展開入力の正規化
  - 非対話モードでの必須項目チェック
- 実行フェーズでは外部依存呼び出しを限定し、判定分岐を削減する。

## 検証
- 引数解析は純粋関数としてユニットテストを追加。
- 認証実行系は既存統合テスト + mock ベース検証で回帰確認。
