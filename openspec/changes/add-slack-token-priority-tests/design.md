## 目的
`SLACK_TOKEN` 優先のトークン解決をテストで固定し、仕様と実装の一致を保証する。

## 設計方針
- **純関数化**: 環境変数の値と token store を引数で受ける解決関数を追加し、ユニットテストで優先順位を検証できるようにする。
- **優先順位の維持**: `--token-type` / `default_token_type` の扱いは現行ルールのままにする。
- **副作用隔離**: `std::env::var` の呼び出しは呼び出し側で行い、テスト対象関数では値を注入する。

## テスト戦略
- `SLACK_TOKEN` が存在する場合に token store の値より優先されることを検証する。
- `SLACK_TOKEN` が未設定の場合に token store が使われることを検証する。
- 既存の明示指定（`--token-type` / `default_token_type`）の振る舞いは回帰テストで保護する。
