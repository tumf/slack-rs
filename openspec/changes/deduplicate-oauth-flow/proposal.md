# 変更提案: OAuth ログインフロー重複を一本化する

## 背景
- `auth::login` と `perform_oauth_flow` に OAuth 実行手順の重複が存在し、修正時に二重更新が必要になっている。
- 認証フローの保守で差分漏れが発生しやすい。

## 目的
- OAuth 実行手順を単一路線に統合し、重複実装を削減する。
- トークン保存・プロフィール保存の既存挙動は維持する。

## 変更範囲
- 認証コード取得〜トークン交換〜レスポンス解釈の共通フロー化方針を定義する。
- 既存ログイン経路（標準/拡張）で同一コアを利用することを要件化する。

## 非目的
- OAuth パラメータや Slack 側設定要件は変更しない。
