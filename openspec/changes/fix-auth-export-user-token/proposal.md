## 変更提案: auth export のユーザートークン欠落修正

### 背景
`auth export` は token store から `team_id:user_id` のキーのみ取得しており、`team_id:user_id:user` に保存されるユーザートークンがエクスポートされない。
その結果、import 後にユーザートークンが復元されず、ユーザートークン前提の操作が失敗する。

### 目的
- `auth export` でユーザートークンが存在する場合は暗号化ペイロードへ含める
- `auth import` でユーザートークンを正しいキーに復元する
- 既存のエクスポート形式との後方互換性を維持する

### 範囲
- Export/Import フォーマットの拡張（ユーザートークンを任意フィールドとして追加）
- export/import 実装とテストの更新

### 非対象
- OAuth フローや token store の保存仕様の変更
- 既存トークンの再発行や自動移行
