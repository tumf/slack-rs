# 提案: auth login の内部構造整理

## 背景
`auth login` の実装は長い関数と多数の分岐を含み、変更時の影響範囲が大きくなっています。読みやすさとテスト容易性の改善が必要です。

## 目的
- `auth login` の内部ロジックを段階的な構成に分割する
- 既存の CLI 振る舞いと OAuth 仕様を維持する

## 範囲
- `login_with_credentials` 周辺の引数解決・設定解決・フロー開始をヘルパー関数に分割
- 引数/設定をまとめる構造体（例: `LoginConfig`）の導入

## 非目標
- OAuth フロー仕様の変更
- 新しいフラグや入力の追加

## 影響・リスク
- 設定解決順序の変更による回帰
- 入力プロンプト/デフォルト値の取り違え

## 受け入れ基準
- 既存の `auth login` コマンドが同じ引数で動作する
- 対話入力・デフォルト値・エラー文言が変わらない

## 依存関係
- なし
