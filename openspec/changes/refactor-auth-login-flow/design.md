# 設計メモ: auth login の内部構造整理

## 方針
- 既存の OAuth 仕様と CLI 挙動を保持する
- 長い関数は「入力解決」「設定構築」「フロー実行」に分離する

## 変更概要
- `LoginConfig` のような構造体で引数を集約
- 構成解決を `resolve_*` ヘルパーに分割
- OAuth 実行は単一の関数に集約

## 代替案
- 単純な関数分割のみ
  - 引数のまとまりが不明確で、呼び出し順序の管理が難しい

## 影響範囲
- `src/auth/commands.rs`
- `src/oauth/*`（呼び出しインターフェースは維持）

## 互換性
- `auth login` の入力/出力/エラーは従来通り
