## 目的
ガイダンス表示の仕様をユニットテストで固定し、stderr への出力依存を減らして TDD を可能にする。

## 設計方針
- **純関数化**: `ApiCallResponse` からガイダンス文字列を生成する関数を用意し、表示はその結果を stderr に出すだけにする。
- **後方互換**: JSON 出力や既存の表示タイミングは変更しない。
- **決定的なフォーマット**: `Error:`/`Cause:`/`Resolution:` のラベルを含む形式を維持する。

## テスト戦略
- 既知エラー（例: `missing_scope`）でガイダンス文字列が返ることを検証する。
- 未知エラーや `ok=true` の場合は `None` が返ることを検証する。
- stderr のキャプチャに依存せず、純関数の戻り値で検証する。
