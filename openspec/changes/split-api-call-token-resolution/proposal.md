# 変更提案: run_api_call のトークン解決責務を分離する

## 背景
- `src/cli/handlers.rs` の `run_api_call` は引数解析・プロファイル解決・トークン解決・フォールバック・出力まで責務が集中している。
- とくにトークン解決分岐が複雑で、仕様維持のまま変更する難易度が高い。

## 目的
- トークンタイプ決定とトークン取得ロジックを専用関数へ分離し、意図を明確化する。
- `SLACK_TOKEN` 優先や明示指定時の失敗条件など既存仕様を固定化する。

## 変更範囲
- `run_api_call` からトークン解決責務を抽出するリファクタ方針を定義する。
- 仕様差分として API 呼び出し時のトークン選択規約維持を明示する。

## 非目的
- API レスポンス形式や Slack API 呼び出し方式の変更は行わない。
