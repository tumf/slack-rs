# 提案: CLI ハンドラー分離

## 背景
`src/main.rs` に複数のコマンド処理が集中しており、関数が巨大化しています。読みやすさ・変更安全性・テスト容易性に影響が出ています。

## 目的
- コマンド処理を専用ハンドラーへ分離し、`main.rs` をディスパッチ中心に簡素化する
- 既存の CLI 挙動を維持しながら、将来の変更容易性を高める

## 範囲
- `auth`/`api`/`auth export`/`auth import` などの処理をハンドラーモジュールに移動
- `main.rs` は引数解析の入口とハンドラー呼び出しに限定

## 非目標
- CLI 仕様や出力形式の変更
- 新規機能の追加

## 影響・リスク
- ルーティングや引数処理の移動による回帰リスク
- 既存テストの更新が必要になる可能性

## 受け入れ基準
- 既存の CLI コマンドが同じ引数で実行でき、出力が変わらない
- `main.rs` の主要処理がハンドラー呼び出しに整理される

## 依存関係
- なし
