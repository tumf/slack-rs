# 設計: スペース区切りオプションの解釈

## 方針
- 既存の `--option=value` 形式を優先し、後方互換を保つ
- `--option value` 形式は次トークンが存在し、かつ `-` で始まらない場合のみ値として扱う
- 値が欠落している場合は従来通り「未指定」として扱い、既存のエラーハンドリングを崩さない
- 複数指定オプション（`--filter` など）は混在形式を許可する

## 変更対象
- `src/cli/mod.rs` のオプション取得ヘルパー
  - `get_option` / `get_all_options` を拡張、または新関数を追加して差し替える
- 既存のコマンド側ロジックは極力触らず、ヘルパー側で吸収する

## 例
- `--filter=is_private:true` → 既存と同様に値を取得
- `--filter is_private:true` → 追加対応で値を取得
- `--format table --raw` → `--format` は `table` を取得、`--raw` はフラグとして残る
- `--format --raw` → `--format` の値なし扱い（従来同様に無視/既定値）

## 影響とリスク
- 値の欠落時に新たなエラーを出さないため、互換性リスクは低い
- ただし `--option` の直後に `-` で始まる値（例: `--sort -name`）は値とみなされない
  - 既存の CLI 使用例に該当がないため影響は限定的
