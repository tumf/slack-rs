# 設計メモ: export/import 引数パース共通化

## 方針
- 既存の CLI 引数仕様を維持する
- 共有ロジックは 1 つの構造体/パーサーに集約する

## 変更概要
- `ExportImportArgs` のような共通構造体を作成
- `parse_export` と `parse_import` を提供して差分のみ扱う

## 代替案
- 既存の重複コードを部分的に切り出す
  - 再利用性が低く、再度の重複が起きやすい

## 影響範囲
- `src/main.rs`（または export/import 実装モジュール）

## 互換性
- 既存のフラグ/入力/出力は変更しない
