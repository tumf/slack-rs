# 変更提案: メンション解決とユーザーキャッシュの追加

## 背景
現状の CLI はメッセージ本文内の `<@U123456>` 形式をそのまま表示しており、ユーザー名の解決が行われていない。

## 目的
- メンションをユーザー名に解決する仕組みを提供する
- `users.list` を用いたキャッシュで API 呼び出し回数を抑える

## 対象範囲
- ユーザー情報キャッシュの保存と更新
- メンション解析と置換
- `users` コマンドの拡張（キャッシュ更新、メンション解決）

## 対象外
- チャンネル/グループ/特殊メンション（`<#C...>` / `<!here>` など）の解決
- 既存コマンドの出力 JSON への自動挿入（例: `api call` の自動置換）
- 実際の Slack ワークスペースへの依存した検証

## 依存関係
- 既存の wrapper-commands 仕様
- `users:read` スコープ

## 成功条件
- キャッシュがない/期限切れの場合に `users cache-update` で更新できる
- `users resolve-mentions` が `<@U...>` を `@display_name` に置換できる
- キャッシュ未命中時は元のメンション表記を保持する
